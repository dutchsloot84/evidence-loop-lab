# 90 Day Capability Map

## Purpose

The first 90 days should build capability in layers, not scatter across many interesting directions.

The sequence is:

1. Learn claim/evidence reconciliation.
2. Apply it to FieldIQ-shaped pilot proof.
3. Add governance and knowledge-loop practices around the working loops.

## Days 1-30: Release Evidence Reconciler

Active lab:

[[Tiny Release Evidence Reconciler]]

Capability:

Check release-readiness claims against observable evidence.

Core loop:

claim -> evidence -> risk -> unsupported claims -> next evidence -> markdown report

Target artifacts:

- loop spec
- 20-example synthetic eval set
- markdown readiness report template
- tiny local workflow or manual procedure
- scorecard
- failure-modes note
- 5-minute teach-back

## Days 31-60: FieldIQ Pilot Proof Lab

Capability:

Define and test the sacred FieldIQ pilot loop using mock or sanitized examples.

Core loop:

login -> visit capture -> generated summary -> review -> repeat

Target artifacts:

- pilot proof loop spec
- mock visit-capture examples
- generated-summary quality rubric
- review scorecard
- failure-modes note
- pilot proof report template
- teach-back demo

Boundary:

This is FieldIQ pilot proof, not general AI architecture exploration.

## Days 61-90: Agent Governance and Knowledge Loop

Capability:

Document approval boundaries, stop conditions, review logs, and knowledge-graph maintenance practices around loops that already work.

Core loops:

agent task -> evidence -> approval boundary -> output -> review log

raw source -> linked synthesis -> neighbor updates -> weekly review

Target artifacts:

- agent governance playbook
- approval-boundary checklist
- stop-condition template
- Obsidian knowledge-loop spec
- weekly review improvements
- teach-back demo tying all labs together

## 90-Day Outcome

By day 90, Evidence Loop Lab should have:

- one reusable release-evidence playbook
- one FieldIQ pilot-proof loop spec
- one agent governance checklist
- one Obsidian knowledge-loop practice
- at least three teach-back artifacts
- a clear decision about which loop deserves deeper productization
