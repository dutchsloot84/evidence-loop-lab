# Project Charter

## Project

Evidence Loop Lab

## Purpose

Evidence Loop Lab is a local-first learning and proof lab for evidence-based AI workflows. Its purpose is to build practical skill in agentic engineering by creating tiny, repeatable loops that turn status claims into checked outputs, evals, playbooks, and teach-back artifacts.

This is an umbrella project for:

- Karpathy-style learning by building
- release readiness and evidence quality
- agent loops and governance
- evals and scorecards
- FieldIQ pilot-proof thinking
- Rovo and release-intelligence workflows
- Obsidian knowledge-graph maintenance
- teach-back demos

## North Star

Turn status claims into evidence-backed, repeatable loops.

Expanded:

Evidence Loop Lab helps me learn agentic engineering by building tiny loops that check claims against evidence, produce scored outputs, document failure modes, and become reusable playbooks for real projects.

## What This Is

Evidence Loop Lab is:

- a learning lab
- a proof lab
- a reusable workflow library
- a place to practice eval-driven AI work
- a place to convert ambiguous claims into evidence-backed reports
- a place to produce concrete artifacts every sprint

## What This Is Not

Evidence Loop Lab is not:

- a general second brain
- a broad AI architecture playground
- a production SaaS product
- a dumping ground for work notes
- a repository for sensitive company data
- a replacement for FieldIQ pilot proof
- a dashboard project before markdown loops work

## Core Constraint

Every active experiment must produce at least one concrete artifact:

- a tiny working build
- an eval or scorecard
- a playbook or loop spec
- a demo or teach-back artifact

## Initial Active Lab

The first active lab is:

[[Tiny Release Evidence Reconciler]]

Reason:

Release evidence reconciliation teaches the most transferable pattern: compare a readiness claim against observable evidence, identify unsupported claims, assign risk, and request the next evidence needed.

This transfers to:

- Rovo release intelligence
- FieldIQ pilot proof
- Codex workflow governance
- eval design
- Obsidian synthesis practices
- career-capital teach-backs
