# Week 1 Sprint Plan

## Sprint

Tiny Release Evidence Reconciler

## Time Budget

Target: about 8 hours

## Week 1 Goal

Create the foundation for the first lab without overbuilding.

By the end of Week 1, the lab should have:

- a project charter
- operating principles
- a lab artifact standard
- decision rules
- a weekly review template
- a lab brief
- a loop spec
- a report template
- a clear plan for the 20-example mock eval set

## Work Slices

### Slice 1: Foundation Notes

Deliver:

- [[Project Charter]]
- [[North Star and Principles]]
- [[Lab Artifact Standard]]
- [[Decision Rules]]
- [[Weekly Review Template]]
- [[90 Day Capability Map]]

Done when:

- the notes define the purpose, boundaries, and decision rules for the lab

### Slice 2: First Lab Definition

Deliver:

- [[Lab Brief]]
- [[Loop Spec]]
- [[Report Template]]

Done when:

- the first lab has a clear input, output, readiness labels, and human approval boundary

### Slice 3: Mock Data Plan

Deliver:

- list of 20 planned synthetic release packets
- expected readiness label for each
- failure mode category for each

Done when:

- the eval set can be created without using real or sensitive release data

### Slice 4: Week 1 Review

Deliver:

- one weekly review record in `07 Weekly Reviews`

Done when:

- the next smallest useful slice is named

## Parallelizable Work

These can be worked on in parallel after the lab brief is stable:

- generating synthetic release packets
- writing expected labels and rationales
- drafting the report template
- drafting the scoring rubric
- drafting the teach-back outline

## Week 1 Stop Condition

Stop Week 1 when the loop is clear enough to generate the 20-example eval set.

Do not start coding until:

- the eval examples exist
- the report format exists
- the scoring rubric exists
