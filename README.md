# Evidence Loop Lab

Evidence Loop Lab is a local-first learning and proof lab for evidence-based AI workflows.

North star:

Turn status claims into evidence-backed, repeatable loops.

The project is designed for practical learning: each active lab should produce a concrete artifact such as a tiny working build, an eval or scorecard, a loop spec, a playbook, or a teach-back demo.

## Current Active Lab

The first active lab is `Tiny Release Evidence Reconciler`.

Goal:

Turn a release-readiness claim into an evidence-backed markdown report by checking the claim against observable evidence, naming unsupported claims, assigning risk, and requesting the next evidence needed.

## Structure

- `00 Operating System/`: project charter, principles, artifact standards, review loops, and decision rules
- `01 Release Intelligence Lab/`: first lab for release evidence reconciliation
- `02 Field Operations Pilot Proof Lab/`: future pilot-proof loop work using mock or sanitized examples
- `03 Agent Governance Lab/`: future approval-boundary and stop-condition work
- `04 Obsidian Knowledge Loop Lab/`: future knowledge-graph maintenance practices
- `05 Personal Learning Labs/`: future learning experiments
- `06 Teach-Backs and Demos/`: short demo and explanation artifacts
- `07 Weekly Reviews/`: weekly review records and lab decisions

## Data Rule

Use synthetic or sanitized data by default.

Do not include customer names, employee names, internal project names, repo names, ticket IDs, PR links, exact ticket or PR titles, proprietary strategy, or sensitive business context.
